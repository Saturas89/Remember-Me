# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities responsibly and privately.

Do not open public GitHub issues for security-sensitive topics.

Instead, contact:

security@storyhold.app

Please include:

- Description of the issue
- Reproduction steps
- Potential impact
- Suggested mitigation if available

## Scope

This includes:

- Authentication
- Personal data handling
- Encryption
- Sharing flows
- AI integrations
- Third-party integrations
- Offline/local storage

## Our principles

Storyhold is built with a strong focus on:

- Privacy-first architecture
- Minimal data collection
- Transparency
- User ownership
- Exportability

We appreciate responsible disclosure and security research.
