# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest (`main`) | Yes |
| older branches | No |

We only provide security fixes for the current `main` branch. Please make sure you are running the latest version before reporting a vulnerability.

## Reporting a Vulnerability

Please report security vulnerabilities responsibly and privately.

**Do not open public GitHub issues for security-sensitive topics.**

Instead, contact us at:

**storyholdapp@gmail.com**

Please include:

- Description of the issue
- Reproduction steps
- Potential impact
- Suggested mitigation if available

We will acknowledge your report within **72 hours** and provide a more detailed response within 7 days, including our assessment and planned timeline for a fix.

## Disclosure Timeline

We follow a coordinated disclosure process:

1. Report received and acknowledged within 72 hours
2. Issue assessed and severity determined within 7 days
3. Fix developed and tested
4. Fix released to `main`
5. Public disclosure after the fix is available (typically 90 days after the initial report, or sooner if agreed with the reporter)

We ask that you give us reasonable time to address the issue before public disclosure.

## Scope

This security policy covers:

- Authentication and authorization
- Personal data handling and storage
- Encryption (at rest and in transit)
- Sharing flows and access controls
- AI integrations
- Third-party integrations
- Offline/local storage
- Export and import functionality

Issues outside this scope (e.g., third-party services not under our control) should be reported directly to the affected vendor.

## Our principles

Storyhold is built with a strong focus on:

- Privacy-first architecture
- Minimal data collection
- Transparency
- User ownership of their data
- Exportability

We appreciate responsible disclosure and security research, and we will publicly credit researchers who responsibly disclose vulnerabilities (unless they prefer to remain anonymous).
